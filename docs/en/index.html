Description

NullOut is a software library that provides an alternative solution to eliminate nulls. in projects that are in production and in which the option to eliminate nulls from The root, that is, from the Model layer, is more tedious and requires high consumption of manual resources. of work or dependence on other equipment. NullOut is focused on removing conditional branches occasioned by the ternary operator “?” in Kotlin, within the coverage analysis of the SonarQube tool.

Installation

Steps to install and configure the project.

In project gradle add the following lines:


                        plugins {
id("com.google.devtools.ksp") version "KotlinVersion-VersionKSP" apply false
}

buildscript {
    repositories {
        ...
        maven(url = "https://jitpack.io")
    }
    ...
}

in app gradle add the following lines:


                        
plugins{
	id("com.google.devtools.ksp")
}
...
dependecies {
	implementation("com.github.brunonavarro:DecoradorNullOut:1.0.0-beta5")
	implementation("com.github.brunonavarro:NullOutLib:1.0.0-beta1")
	ksp("com.github.brunonavarro:ProcesadorNullOutLib:1.0.0-beta2")
}

Use

Instructions on how to use the project.

These generic functions should not be implemented for attributes of data classes that at least do not They initialize their attributes to null for example:

Not Supported


                        
data class User(
   val userName: String,
   val password: String
)

Supported


                        
data class User(
   val userName: String? = null,
   val password: String? = null
)

The following interface must be implemented in all data classes:


                        
interface WithDefault

In addition, the following annotation must also be added:


                        
annotation NullOutEntity

Resulting in the data class:


                        
@NullOutEntity
data class User(
   val userName: String? = null,
   val password: String? = null
): WithDefault

Functional references

notNull functions for primitive variables:

Supported primitive variables are: (String, Int, Double, Boolean)


                            fun String?.getOrDefault(value: String? = null): String
fun Int?.getOrDefault(value: Int? = null): Int
fun Double?.getOrDefault(value: Double? = null): Double
fun Boolean?.getOrDefault(value: Boolean? = null): Boolean
fun Number?.getOrDefault(value: Number? = null): Boolean

All these functions are responsible for receiving the real value of the attribute and If null, it returns a default value, which can be indicated or failing that they would be:


String -> “” Double -> 0.0 Int -> 0 Boolean -> false

notNull functions for Pair or Map variables:

Supported primitive variables are: (String, Int, Double, Boolean)


                            fun  Pair?.getPairOrDefault(default: Pair? = null): Pair where A: WithDefault, B: WithDefault
fun  Pair?.getPairOrDefault(default: Pair? = null): Pair where B : Any, A : Any
fun  Pair?.getPairOrDefault(default: Pair? = null): Pair where A: WithDefault, B: WithDefault
fun  Pair?.getPairOrDefault(default: Pair? = null): Pair where A : Any, B : Any
fun  Pair?.getPairOrDefault(default: Pair? = null): Pair where A : Any, B : Any
fun  Pair?.getPairOrDefault(default: Pair? = null): Pair where A : WithDefault, B : WithDefault

fun  Map?.getMapOrDefault(default: Map? = null): Map
fun  Map?.getMapOrDefault(default: Map? = null): Map where K: WithDefault, V: WithDefault
fun  Map?.getMapOrDefault(default: Map? = null): Map
fun  Map?.getMapOrDefault(default: Map? = null): Map where K: WithDefault, V: WithDefault
fun  Map?.getMapOrDefault(default: Map? = null): Map
fun  Map?.getMapOrDefault(default: Map? = null): Map where K: WithDefault, V: WithDefault

All these functions are responsible for receiving the real value of the attribute and If any is null, it returns a default value, which can be indicated or, failing that, an instance is created according to the type of variable.

NotNull functions for object variables:


                            fun  T?.getOrDefault(): T where T: WithDefault
fun  T?.getOrDefault(): T where T: Any

All these functions are responsible for receiving the real value of the attribute and If it is null, it returns a default value, for which an instance is created according to the type of variable.

NotNull functions for array variables:


                            fun  Array?.getArrayOrDefault(): Array where T : WithDefault
fun  Array?.getArrayOrDefault(): Array where T: Any
fun  Array?.getArrayOrDefault(default: Array = arrayOf()): Array where T : WithDefault
fun  Array?.getArrayOrDefault(default: Array = arrayOf()): Array where T: Any

All these functions are responsible for receiving the real value of the attribute and If null, it returns a default value, for which an empty array is created.

NotNull functions for arrayList variables:

fun  ArrayList?.getArrayListOrDefault(default: ArrayList? = null): ArrayList where T : Any
fun  ArrayList?.getArrayListOrDefault(default: ArrayList = arrayListOf()): ArrayList where T : WithDefault
fun  ArrayList?.getArrayListOrDefault(default: ArrayList = arrayListOf()): ArrayList where T : Any

All these functions are responsible for receiving the real value of the attribute and If null, it returns a default value, for which an empty arrayList is created.

NotNull functions for List variables:

All these functions are responsible for receiving the real value of the attribute and If null, it returns a default value, for which an empty List is created.
